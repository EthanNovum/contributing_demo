## 内容类型

我们欢迎以下类型的内容贡献：

1. **AI提示词示例**：各类场景下的有效提示词示例
2. **AI安全研究**：关于AI安全的研究论文、案例分析或漏洞报告
3. **教程与指北**：使用AI技术的教程或最佳实践指北

## 内容贡献流程

### 1. 提交新内容

要提交新内容，请按照以下步骤操作：

1. Fork我们的GitHub仓库
2. 在适当的目录创建新的Markdown文件
   - 对于提示词示例：`/pages/prompts/[category]/[name].mdx`
   - 对于安全研究：`/pages/security/[category]/[name].mdx`
3. 按照模板格式编写内容
4. 提交Pull Request

### 2. 更新现有内容

要更新现有内容，请按照以下步骤操作：

1. 找到需要更新的内容文件
2. 进行必要的修改和改进
3. 提交Pull Request，并在描述中说明修改的原因和内容

## 内容格式要求

### 提示词示例模板

```md
# [提示词标题]

## 简介
简要描述提示词的用途和适用场景。

## 提示词
```
[此处插入提示词内容]
```

## 示例输出
```
[此处插入示例输出]
```

## 使用说明
详细解释如何使用该提示词，包括可能的变量和调整方法。

## 贡献者
- [贡献者姓名](GitHub链接)
```

### 安全研究模板

```md
# [研究标题]

## 摘要
简要概述研究内容和主要发现。

## 背景
提供必要的背景信息。

## 方法/分析
详细描述研究方法或分析过程。

## 结果
展示研究结果。

## 讨论
讨论结果的含义和潜在影响。

## 参考资料
列出参考文献和资源。

## 贡献者
- [贡献者姓名](GitHub链接)
```

## 内容审核标准

所有提交的内容将根据以下标准进行审核：

1. **准确性**：内容应当准确无误
2. **原创性**：内容应为原创或适当引用
3. **实用性**：内容应具有实际应用价值
4. **清晰度**：内容应表述清晰、结构合理
5. **合规性**：内容不应包含不当信息或侵犯知识产权
